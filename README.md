# gtseq_microhap

Alignment-free microhaplotype genotyping for GT-seq amplicon sequencing data.

gtseq_microhap identifies and genotypes microhaplotypes directly from paired-end FASTQ files generated by GT-seq panels. The pipeline resolves paired-end reads into primer-bounded amplicon sequences, constructs a catalog of observed haplotypes across samples, and performs abundance-based genotype calling under a diploid model.

Unlike conventional pipelines, this workflow does not require read alignment or variant calling, instead leveraging the high read depth typical of GT-seq assays.

------------------------------------------------------------
FEATURES
------------------------------------------------------------

Alignment-free microhap genotyping

• Works directly from paired-end FASTQ files  
• No reference genome or BAM files required  
• Only short R1/R2 overlap merging is performed  


Primer-bounded read resolution

• Identifies reads beginning with locus-specific forward primers  
• Confirms reverse primer in the paired read  
• Merges paired reads to reconstruct full amplicon sequences  
• Produces locus-resolved FASTQ files for downstream processing  


Strict allele catalog construction

• Identifies unique amplicon sequences within each sample  
• Retains only the top two alleles per sample under a diploid model  
• Filters background or spurious sequences  
• Aggregates alleles across samples to construct a microhaplotype catalog  


Catalog-based genotype calling

A second pass assigns reads only to catalog alleles.

Genotypes are inferred using A1/A2 read abundance ratios.

Genotype classes:

HOM  – homozygous  
HET  – heterozygous  
NC   – ambiguous call  
LOW  – insufficient depth  

Allele frequencies are calculated from called genotypes only, preventing bias from background reads.


Phased microhaplotype generation

Catalog haplotypes are aligned to identify polymorphic positions.

Outputs include:

• SNP positions  
• indel variation  
• phased microhaplotype sequences  

These phased haplotypes can be used for downstream analyses including:

• kinship inference  
• parentage analysis  
• population genetics  
• assignment testing  


QC dashboards

Per-locus QC dashboards are automatically generated:

plots/<locus>_dashboard.png

Dashboards include:

• A1 vs A2 read counts  
• Total depth vs %A2  
• Genotype call rate  
• Background allele counts  
• Allele frequency barplots  
• Panel-wide locus balance  

These diagnostics help identify problematic loci or assay imbalance.

QUICK START
------------------------------------------------------------

Inputs


1. Paired-end FASTQ files

Place all FASTQ files in a single directory.

Example:

sample01_R1.fastq.gz  
sample01_R2.fastq.gz  
sample02_R1.fastq.gz  
sample02_R2.fastq.gz  


2. Primer definition file

CSV containing locus primer sequences:

locus,fwd_primer,rev_primer  
NC_061065.1_5347094,ACGT...,TGCA...  
NC_061065.1_2389012,GCTA...,AGTC...  


------------------------------------------------------------
BASIC USAGE
------------------------------------------------------------

python3 gtseq_microhap_catalog_and_call.py \
  --indir ./R1_R2 \
  --primers ./primers.csv \
  --outdir microhap_out


------------------------------------------------------------
PIPELINE STEPS
------------------------------------------------------------

1. Resolve paired-end reads

Reads are scanned for forward primer sequences and confirmed with reverse primers.

Resolved sequences are written to:

resolved_fastqs/resolved_<sample>.fastq.gz


2. Build allele catalog

Unique amplicon sequences are identified across samples.

Outputs:

microhap_catalog.csv  
microhap_consensus.fasta  


3. Genotype samples

Reads are assigned to catalog alleles.

Outputs:

microhap_genotypes.tsv  
microhap_a2_metrics.tsv  


4. Generate QC dashboards

Per-locus diagnostic plots:

plots/<locus>_dashboard.png



------------------------------------------------------------
IMPORTANT OPTIONS
------------------------------------------------------------

Amplicon size filters

--min-amplicon  
--max-amplicon  

Defines acceptable stitched amplicon length.


Minimum genotype depth

--min-depth  

Minimum reads required for genotype calling.


A2 decision thresholds

--a2-lo  
--a2-hi  

Controls HOM / HET / NC classification boundaries.


Parallel processing

--workers  

Number of worker processes used for FASTQ resolving.


Force rebuilding resolved FASTQs

--force  

Re-run the read resolution stage even if outputs already exist.


Use an existing allele catalog

--use-catalog microhap_catalog.csv  

Allows genotype calling using a previously generated catalog without rebuilding it.

Example dataset
-----------------------------------------------------------

A trimmed demonstration dataset used in the manuscript is available at Zenodo:

https://doi.org/10.5281/zenodo.19069551

The dataset contains paired-end FASTQ files from 96 delta smelt individuals
genotyped with a 410-locus GT-seq panel. Reads were subsampled to ~150,000
read pairs per sample to reduce download size while preserving locus
representation and genotype structure.


------------------------------------------------------------
HOW THIS DIFFERS FROM CONVENTIONAL SNP PIPELINES
------------------------------------------------------------

Traditional SNP pipelines typically involve:

FASTQ → alignment → BAM → variant calling → VCF

This pipeline instead uses a direct amplicon-based approach:

FASTQ → primer-bounded reads → haplotype catalog → genotype calls

Advantages include:

• no reference genome required  
• faster analysis  
• direct recovery of phased haplotypes  
• robust performance with high-depth targeted sequencing
